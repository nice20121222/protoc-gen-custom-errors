# protoc-gen-custom-errors

`protoc-gen-custom-errors` generates Go error helpers from error enums defined in Protocol Buffers. The generated errors implement the `GRPCStatus()` interface, support conversion between gRPC and HTTP status codes, and expose a business reason and metadata through error details.

An error is represented as JSON like this:

```jsonc
{
  // The error code is consistent with HTTP status and can be converted into a gRPC status.
  "code": 500,
  // The error reason is the business-specific error code.
  "reason": "USER_NOT_FOUND",
  // The error message is readable and can be shown to users.
  "message": "invalid argument error",
  // Metadata carries additional extensible error information.
  "metadata": {
    "code": "101"
  }
}
```

## Installation

```sh
go install github.com/nice20121222/protoc-gen-custom-errors@latest
```

Ensure the Go binary directory is on your `PATH` so that `protoc` can find the plugin.

## Error definition

Define an error enum and configure its default values with options from `errors/errors.proto`. Individual enum values can override the HTTP status code, numeric business code, and message.

```proto
syntax = "proto3";

package api.kratos.v1;

import "errors/errors.proto";

// Define the Go package name for generated source code.
option go_package = "kratos/api/helloworld;helloworld";
option java_multiple_files = true;
option java_package = "api.helloworld";

enum ErrorReason {
  // Set the default HTTP error code.
  option (errors.default_code) = 500;

  // Set the default numeric business error code.
  option (errors.default_num_code) = 500;

  // Set the default error message.
  option (errors.default_message) = "默认错误";

  // 没有权限 132001
  NO_PERMISSION = 0 [(errors.code) = 403, (errors.num_code) = 132001, (errors.message) = "没有权限"];
}
```

## Generation

Run `protoc` with the Go generator and this plugin:

```sh
protoc --proto_path=. \
  --proto_path=./third_party \
  --go_out=paths=source_relative:. \
  --custom-errors_out=paths=source_relative:. \
  $(API_PROTO_FILES)
```

## Generated output

After successful generation, a Go file is created in the `api/helloworld` directory. For the definition above, the generated helpers are equivalent to:

```go
// Code generated by protoc-gen-custom-errors. DO NOT EDIT.

package helloworld

import (
	fmt "fmt"
	errors "github.com/go-kratos/kratos/v2/errors"
)

// This is a compile-time assertion to ensure that this generated file
// is compatible with the kratos package it is being compiled against.
const _ = errors.SupportPackageIsVersion1

// 没有权限, 132001
func IsNoPermission(err error) bool {
	if err == nil {
		return false
	}
	e := errors.FromError(err)
	return e.Reason == ErrorReason_NO_PERMISSION.String() && e.Code == 403
}

// 没有权限, 132001
func ErrorNoPermission(format string, args ...interface{}) *errors.Error {
	if format == "" {
		return errors.New(403, ErrorReason_NO_PERMISSION.String(), "没有权限").
			WithMetadata(map[string]string{"code": "132001"})
	}
	return errors.New(403, ErrorReason_NO_PERMISSION.String(), fmt.Sprintf(format, args...)).
		WithMetadata(map[string]string{"code": "132001"})
}
```
